Code Test

Implement RESTful Services, which provide the solution of the following tasks, given the sample paragraphs. Also protect the services with Spring Security.

Task 1

Search the following texts, which will return the counts respectively.

As a user, I will be able to execute the following curl command and returning results in the json format:

Sample Request:

curl http://host/counter-api/search -H "authorization: Basic b3B0dXM6Y2FuZGlkYXRlcw==" -d '{"searchText":["Duis", "Sed", "Donec", "Augue", "Pellentesque", "123"]}' -H "Content-Type: application/json" -X POST

Result in JSON:

{
  "counts": [
    {
      "Duis": 11
    },
    {
      "Sed": 16
    },
    {
      "Donec": 8
    },
    {
      "Augue": 7
    },
    {
      "Pellentesque": 6
    },
    {
      "123": 0
    }
  ]
}

Note: Basic authorization header is using Base64 format with UTF-8 CharSet

Task 2

Provide the top 20 (as Path Variable) Texts, which has the highest counts in the Sample Paragraphs provided.

As a user, I will be able to execute the following curl command and expecting result in csv format. I will be able to put 10, 20, 30 or 5 as the top listing.

Sample Request:

curl http://host/counter-api/top/20 -H "authorization: Basic b3B0dXM6Y2FuZGlkYXRlcw==" -H "Accept: text/csv"

As an example of the result if I put /top/5: 

text1|100
text2|91
text3|80
text4|70
text5|60

Implementation

Task 1

After analysis, below points listed:

  • search text is case-insensitive

For example, "Sed" (case-sensitive) found 4 times while "sed" (case-insensitive) found 16 times. Since the requirement indicates "Sed" has been found 16 times, so the search function should treat text as case-insensitive.

  • Http Authorization

After decoded "Basic authorization header", I got username "optus" and password "canidates". For security purpose, I put them in application.properties rather than hard coding them in WebSecurityConfig.

  • Sample Paragraphs

I put the sample paragraphs into a file (sample.txt) and store it as a resource file in src/main/resources directory.

Task 2

  • highest counts with case-insensitive rule

Current requirement doesn't indicate this, however, to stay in the same page of Task 1, I set the function with case-insensitive rule.

  • min and max of top listing

As requirement mentioned, top listing can be varied, so I set a range (min~max) for path variable validation. These settings can be found in application.properties
